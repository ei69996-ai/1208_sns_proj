# 스크린 리더 테스트 가이드

이 문서는 Instagram Clone SNS 프로젝트의 스크린 리더 호환성 테스트를 위한 가이드입니다.

## 테스트 도구

### Windows
- **NVDA** (무료): https://www.nvaccess.org/
- **JAWS** (유료): https://www.freedomscientific.com/

### macOS
- **VoiceOver** (내장): 시스템 설정 → 접근성 → VoiceOver

### 브라우저
- Chrome (NVDA, JAWS)
- Safari (VoiceOver)
- Firefox (NVDA, JAWS)

## 테스트 시나리오

### 시나리오 1: 홈 피드 탐색

#### 1-1. 페이지 로드
1. 홈 페이지 (`/`) 접속
2. 스크린 리더가 페이지 제목을 읽는지 확인
3. "Instagram" 로고 또는 제목이 읽히는지 확인

#### 1-2. 사이드바 네비게이션 (Desktop/Tablet)
1. Tab 키로 사이드바 메뉴 항목 이동
2. 각 메뉴 항목이 올바르게 읽히는지 확인:
   - "홈" (Home 아이콘)
   - "검색" (Search 아이콘)
   - "만들기" (Plus 아이콘)
   - "프로필" (User 아이콘)
3. 현재 페이지가 하이라이트되어 있는지 확인

#### 1-3. 게시물 카드 탐색
1. Tab 키로 게시물 카드로 이동
2. 게시물 정보가 올바르게 읽히는지 확인:
   - 사용자명
   - 게시물 이미지 (alt 텍스트)
   - 좋아요 수
   - 캡션
   - 댓글 미리보기
3. 액션 버튼이 올바르게 읽히는지 확인:
   - "좋아요" / "좋아요 취소"
   - "댓글"
   - "공유"
   - "저장"

#### 1-4. 게시물 상세 보기
1. 게시물 이미지에 Enter 키로 모달 열기
2. 모달이 열렸다는 것을 스크린 리더가 알리는지 확인
3. 모달 내부 요소 탐색:
   - 사용자명
   - 이미지
   - 좋아요 수
   - 캡션
   - 댓글 목록
4. ESC 키로 모달 닫기

### 시나리오 2: 게시물 작성

#### 2-1. 게시물 작성 모달 열기
1. 사이드바 또는 하단 네비게이션에서 "만들기" 클릭
2. 모달이 열렸다는 것을 스크린 리더가 알리는지 확인
3. "새 게시물 만들기" 제목이 읽히는지 확인

#### 2-2. 이미지 선택
1. 파일 선택 영역으로 포커스 이동
2. "이미지 선택" 버튼이 읽히는지 확인
3. Enter 또는 Space 키로 파일 선택 대화상자 열기
4. 이미지 선택 후 미리보기가 표시되는지 확인

#### 2-3. 캡션 입력
1. 캡션 입력 필드로 포커스 이동
2. "캡션" 라벨이 읽히는지 확인
3. 텍스트 입력
4. 글자 수 카운터가 읽히는지 확인 (예: "100/2200")

#### 2-4. 게시물 업로드
1. "공유" 버튼으로 포커스 이동
2. 버튼 상태가 읽히는지 확인 (활성화/비활성화)
3. 업로드 중 상태가 읽히는지 확인 ("업로드 중...")
4. 성공/실패 메시지가 읽히는지 확인

### 시나리오 3: 프로필 페이지 탐색

#### 3-1. 프로필 페이지 접속
1. 사이드바 또는 하단 네비게이션에서 "프로필" 클릭
2. 프로필 페이지로 이동
3. 프로필 정보가 올바르게 읽히는지 확인:
   - 사용자명
   - 프로필 이미지 (대체 텍스트)
   - 게시물 수 ("게시물 X개")
   - 팔로워 수 ("팔로워 X명")
   - 팔로잉 수 ("팔로잉 X명")

#### 3-2. 게시물 그리드 탐색
1. Tab 키로 게시물 그리드로 이동
2. 각 게시물 이미지가 올바르게 읽히는지 확인 (alt 텍스트)
3. 게시물 클릭 시 상세 모달 열기

#### 3-3. 팔로우 버튼 (다른 사람 프로필)
1. "팔로우" 또는 "팔로잉" 버튼으로 포커스 이동
2. 버튼 상태가 올바르게 읽히는지 확인
3. 버튼 클릭 시 상태 변경이 읽히는지 확인

### 시나리오 4: 댓글 작성

#### 4-1. 댓글 입력
1. 게시물 카드 또는 모달에서 댓글 입력 필드로 포커스 이동
2. "댓글 달기..." placeholder가 읽히는지 확인
3. 텍스트 입력
4. Enter 키로 댓글 제출
5. Shift+Enter로 줄바꿈 가능한지 확인

#### 4-2. 댓글 목록
1. 댓글 목록이 올바르게 읽히는지 확인:
   - 사용자명
   - 댓글 내용
   - 삭제 버튼 (본인 댓글만)

## 권장사항

1. **로딩 상태 알림**
   - "로딩 중..." 메시지가 스크린 리더에 읽히는지 확인
   - `aria-live="polite"` 또는 `aria-busy="true"` 사용 고려

2. **에러 메시지**
   - 에러 메시지가 스크린 리더에 읽히는지 확인
   - `role="alert"` 또는 `aria-live="assertive"` 사용 고려

3. **성공 메시지**
   - 성공 메시지가 스크린 리더에 읽히는지 확인
   - `aria-live="polite"` 사용 고려

4. **포커스 관리**
   - 모달 열림 시 첫 번째 상호작용 요소로 포커스 이동
   - 모달 닫힘 시 이전 포커스 위치로 복귀

## 참고

- 시맨틱 HTML 태그 사용 확인 (`<nav>`, `<main>`, `<article>` 등)
