# 터치 인터랙션 테스트 가이드

이 문서는 Instagram Clone SNS 프로젝트의 터치 인터랙션 테스트를 위한 가이드입니다.

## 테스트 환경

### 권장 도구
- Chrome DevTools Device Mode (터치 이벤트 시뮬레이션)
- 실제 모바일 디바이스 (가장 정확한 테스트)
- Firefox Responsive Design Mode

### 테스트 디바이스 (선택사항)
- iPhone (iOS)
- Android 기기
- iPad (태블릿)

## 테스트 항목

### 1. 더블탭 좋아요 (모바일)

#### 1-1. 기본 동작
- [ ] 이미지 더블탭 시 좋아요 토글
- [ ] 더블탭 간격: 300ms 이내
- [ ] 큰 하트 애니메이션 표시 (fade in/out)
- [ ] 애니메이션 지속 시간: 1초
- [ ] 이미 좋아요한 경우: 더블탭 시 동작하지 않음 (선택사항)

#### 1-2. 컴포넌트
- **파일**: `components/post/PostCard.tsx`
- **함수**: `handleDoubleTap`
- **상태**: `showDoubleTapHeart`

#### 1-3. 테스트 시나리오
1. 홈 피드에서 게시물 이미지 더블탭
2. 큰 하트 애니메이션 확인
3. 좋아요 상태 변경 확인
4. 좋아요 수 증가 확인

### 2. 스와이프 제스처

#### 2-1. 이미지 갤러리 (미래 기능)
- [ ] 좌우 스와이프로 이미지 전환
- [ ] 스와이프 감도: 적절한 거리
- [ ] 애니메이션: 부드러운 전환
- **참고**: 현재는 단일 이미지만 지원

#### 2-2. 모달 닫기 (선택사항)
- [ ] 좌우 스와이프로 모달 닫기
- [ ] 스와이프 거리: 화면의 50% 이상
- [ ] 취소 가능: 스와이프 중단 시 원위치

### 3. 롱프레스 (컨텍스트 메뉴)

#### 3-1. 게시물 카드
- [ ] 이미지 롱프레스: 컨텍스트 메뉴 표시 (선택사항)
- [ ] 롱프레스 시간: 500ms
- [ ] 햅틱 피드백: 진동 (실제 디바이스)

#### 3-2. 댓글
- [ ] 댓글 롱프레스: 삭제 옵션 표시 (선택사항)
- [ ] 본인 댓글만 삭제 가능

### 4. 터치 영역 크기

#### 4-1. 최소 크기
- [ ] 모든 버튼: 최소 44x44px
- [ ] 모든 링크: 최소 44x44px
- [ ] 모든 아이콘: 최소 44x44px (터치 영역)

#### 4-2. 컴포넌트별 확인
- [ ] Sidebar 메뉴 항목
- [ ] BottomNav 아이콘
- [ ] PostCard 액션 버튼 (좋아요, 댓글, 공유, 북마크)
- [ ] 프로필 팔로우 버튼
- [ ] 게시물 작성 버튼
- [ ] 댓글 입력 필드
- [ ] 모달 닫기 버튼

#### 4-3. 간격
- [ ] 버튼 간 최소 간격: 8px
- [ ] 터치 영역 겹침 없음

### 5. 스크롤 제스처

#### 5-1. 수직 스크롤
- [ ] 부드러운 스크롤: 모든 화면
- [ ] 관성 스크롤: 자연스러운 감속
- [ ] 무한 스크롤: 하단 도달 시 자동 로드

#### 5-2. 모달 내부 스크롤
- [ ] PostModal 댓글 영역: 독립적인 스크롤
- [ ] 스크롤 끝 도달: 바운스 효과 (iOS)

### 6. 핀치 줌 (선택사항)

#### 6-1. 이미지 확대
- [ ] 이미지 핀치 줌: 확대/축소 (선택사항)
- [ ] 더블탭 줌: 2배 확대 (선택사항)
- **참고**: 현재는 기본 이미지 표시만 지원

## 테스트 방법

### Chrome DevTools 사용
1. Chrome DevTools 열기 (F12)
2. Device Toolbar 활성화 (Ctrl+Shift+M)
3. 디바이스 선택 (예: iPhone 12 Pro)
4. 터치 이벤트 시뮬레이션 활성화
5. 마우스로 터치 제스처 시뮬레이션

### 실제 디바이스 사용
1. 개발 서버 실행 (`pnpm dev`)
2. 로컬 네트워크 IP 확인
3. 모바일 디바이스에서 접속
4. 실제 터치 제스처 테스트

## 테스트 시나리오

### 시나리오 1: 더블탭 좋아요
1. 홈 피드 열기
2. 첫 번째 게시물 이미지 더블탭
3. 큰 하트 애니메이션 확인
4. 좋아요 수 증가 확인
5. 다시 더블탭하여 좋아요 취소 (선택사항)

### 시나리오 2: 터치 영역 확인
1. 모든 버튼/링크에 손가락으로 터치
2. 터치 영역이 충분한지 확인
3. 실수로 다른 요소를 터치하지 않는지 확인

### 시나리오 3: 스크롤 테스트
1. 홈 피드에서 아래로 스크롤
2. 부드러운 스크롤 확인
3. 하단 도달 시 자동 로드 확인
4. 로딩 상태 확인

### 시나리오 4: 모달 상호작용
1. 게시물 클릭하여 모달 열기
2. 댓글 영역 스크롤
3. 모달 닫기 버튼 터치
4. 배경 터치로 모달 닫기 (선택사항)
