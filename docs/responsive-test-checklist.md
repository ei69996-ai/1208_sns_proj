# 반응형 테스트 체크리스트

이 문서는 Instagram Clone SNS 프로젝트의 반응형 디자인 테스트를 위한 체크리스트입니다.

## 테스트 환경

### 화면 크기

#### Mobile
- **320px** (iPhone SE, 작은 Android 기기)
- **375px** (iPhone 12/13, 표준 모바일)
- **414px** (iPhone 14 Pro Max, 큰 모바일)

#### Tablet
- **768px** (iPad 세로, 작은 태블릿)
- **1024px** (iPad Pro 가로, 큰 태블릿)

#### Desktop
- **1280px** (표준 데스크톱)
- **1920px** (Full HD 데스크톱)

## 테스트 도구

- Chrome DevTools Device Mode
- Firefox Responsive Design Mode
- 실제 디바이스 (선택사항)

## 테스트 항목

### 1. 레이아웃 구조

#### 1-1. Sidebar (사이드바)
- [ ] Desktop (1024px+): 244px 너비, 아이콘 + 텍스트 표시
- [ ] Tablet (768px ~ 1023px): 72px 너비, 아이콘만 표시
- [ ] Mobile (< 768px): 완전히 숨김
- [ ] 메뉴 항목 정렬: 세로로 올바르게 정렬
- [ ] Active 상태: 현재 페이지 하이라이트
- [ ] Hover 효과: 마우스 오버 시 배경색 변경

#### 1-2. BottomNav (하단 네비게이션)
- [ ] Mobile (< 768px): 50px 높이, 5개 아이콘 표시
- [ ] Tablet/Desktop: 숨김
- [ ] 아이콘 정렬: 균등하게 분배
- [ ] Active 상태: 현재 페이지 하이라이트
- [ ] 터치 영역: 각 아이콘 최소 44x44px

#### 1-3. Header (모바일 헤더)
- [ ] Mobile (< 768px): 60px 높이, 로고 + 아이콘 표시
- [ ] Tablet/Desktop: 숨김
- [ ] 아이콘 정렬: 우측 정렬
- [ ] 터치 영역: 각 아이콘 최소 44x44px

### 2. 홈 피드 페이지

#### 2-1. PostCard (게시물 카드)
- [ ] 최대 너비: 630px (중앙 정렬)
- [ ] 배경색: #FFFFFF
- [ ] 이미지 비율: 1:1 정사각형 유지 (모든 화면 크기)
- [ ] 헤더: 프로필 이미지 32px, 사용자명, 시간 표시
- [ ] 액션 버튼: 좋아요, 댓글, 공유, 북마크 아이콘
- [ ] 캡션: 2줄 초과 시 "... 더 보기" 표시
- [ ] 댓글 미리보기: 최신 2개만 표시

#### 2-2. PostFeed (게시물 피드)
- [ ] 배경색: #FAFAFA
- [ ] 중앙 정렬: 최대 너비 630px
- [ ] 게시물 간격: 적절한 간격 유지
- [ ] 무한 스크롤: 하단 도달 시 자동 로드
- [ ] 로딩 상태: Skeleton UI 표시

### 3. 프로필 페이지

#### 3-1. ProfileHeader (프로필 헤더)
- [ ] Desktop: 프로필 이미지 150px
- [ ] Mobile: 프로필 이미지 90px
- [ ] 통계: 게시물 수, 팔로워 수, 팔로잉 수 표시
- [ ] 버튼: "팔로우" / "팔로잉" 버튼
- [ ] 레이아웃: 세로 정렬 (Mobile), 가로 정렬 (Desktop)

#### 3-2. PostGrid (게시물 그리드)
- [ ] Desktop: 3열 그리드
- [ ] Tablet: 3열 그리드
- [ ] Mobile: 3열 그리드 (작은 화면에서는 2열 고려)
- [ ] 이미지 비율: 1:1 정사각형 유지
- [ ] Hover 효과: 좋아요/댓글 수 표시 (Desktop)
- [ ] 터치 영역: 각 이미지 최소 44x44px

### 4. 게시물 작성 모달

#### 4-1. CreatePostModal
- [ ] Desktop: 모달 형식, 최대 너비 2xl
- [ ] Mobile: 전체 화면 또는 큰 모달
- [ ] 이미지 미리보기: 1:1 비율 유지
- [ ] 캡션 입력: 텍스트 영역 적절한 크기
- [ ] 버튼: "공유" 버튼 접근 가능

### 5. 게시물 상세 모달

#### 5-1. PostModal
- [ ] Desktop: 모달 형식, 이미지 50% + 댓글 50%
- [ ] Mobile: 전체 페이지로 전환
- [ ] 이미지: 1:1 비율 유지
- [ ] 댓글 영역: 스크롤 가능
- [ ] 닫기 버튼: 접근 가능

### 6. 텍스트 가독성

- [ ] 텍스트 크기: 모든 화면에서 읽기 가능
- [ ] 줄 간격: 적절한 line-height
- [ ] 텍스트 잘림: 긴 텍스트는 말줄임표 처리
- [ ] 색상 대비: WCAG 2.1 AA 기준 준수

### 7. 버튼 및 링크

- [ ] 터치 영역: 최소 44x44px (모바일)
- [ ] 간격: 버튼 간 적절한 간격
- [ ] 상태 표시: Hover, Active, Focus 상태 명확
- [ ] 접근성: 키보드 접근 가능

### 8. 이미지

- [ ] 비율 유지: 모든 화면에서 1:1 정사각형
- [ ] 로딩: 적절한 로딩 상태 표시
- [ ] 최적화: Next.js Image 컴포넌트 사용
- [ ] Lazy loading: 스크롤 시 로드

### 9. 스크롤 동작

- [ ] 부드러운 스크롤: 모든 화면에서 자연스러운 스크롤
- [ ] 무한 스크롤: 하단 도달 시 자동 로드
- [ ] 스크롤 위치: 새로고침 후 위치 유지 (선택사항)
- [ ] 모달 스크롤: 모달 내부 스크롤 동작 확인

## 테스트 방법

1. Chrome DevTools 열기 (F12)
2. Device Toolbar 활성화 (Ctrl+Shift+M)
3. 각 화면 크기 선택
4. 체크리스트 항목 확인
5. 문제 발견 시 스크린샷 및 설명 기록
